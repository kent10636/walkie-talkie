# 已知问题与限制 (Known Issues & Limitations)

**最后更新**: 2026-06-19

本文档记录当前代码中已知的问题、限制和待改进点。仅供参考，不代表必须立即修复。

## 高优先级（影响核心功能验证）

### 1. 麦克风权限未在运行时请求
- **现象**: App 可能在第一次使用时直接失败，或需要用户手动去设置里授权。
- **当前状态**: Manifest 中已声明权限，但代码中缺少运行时请求逻辑（`ActivityCompat.requestPermissions` 或新版 `PermissionLauncher`）。
- **影响**: 无法在部分设备上立即使用 PTT 功能。
- **建议修复**: 在进入房间前检查并请求 `RECORD_AUDIO` 权限。

### 2. 语音功能尚未端到端验证
- **原因**: 目前没有可用的测试设备。
- **影响**: 无法确认 LiveKit 连接、音频发布、接收是否真正工作。
- **建议**: 获得设备后优先完成双向通话测试。

### 3. 参与者列表不实时更新
- **当前实现**: 仅在连接时加入本地昵称。
- **缺失**: 监听 `room.remoteParticipants` 的变化。
- **影响**: 看不到对方进入/离开。

### 4. 错误处理和连接状态较弱
- **问题**:
  - 网络错误、token 获取失败、LiveKit 连接失败时提示不友好。
  - 没有重连逻辑。
- **影响**: 用户体验差，调试困难。

## 中优先级

### 5. 后端使用纯内存存储
- **现状**: 群组数据存在内存中，进程重启后全部丢失。
- **影响**: 不适合长期测试或演示。
- **计划**: 后续接入 Postgres + sqlc（已在原始计划中）。

### 6. LiveKit 地址硬编码
- **问题**:
  - Android 模拟器用 `10.0.2.2`
  - 真机需要手动改 `ApiClient.kt`
  - 后端返回的 `livekitUrl` 也是写死的
- **建议**: 支持通过配置或启动参数动态传入。

### 7. PTT 手势实现比较基础
- **当前**: 使用 `pointerInput` + `awaitPointerEvent`
- **不足**: 缺少视觉按下反馈、长按判定、防止误触等细节。
- **建议**: 结合 `InteractionSource` 或 `detectTapGestures` 改进。

### 8. 没有处理音频焦点和蓝牙
- **缺失**:
  - `AudioManager` 音频焦点请求
  - 蓝牙 SCO 连接处理
  - 来电/通知打断处理
- **影响**: 专业对讲体验不足。

## 低优先级 / 未来增强

- UI 动画和说话波形可视化
- 群组名称修改、离开群组后端清理
- 简单用户头像或颜色区分
- 推送通知（邀请进入群组）
- 面对面建群功能（原规划中的 stretch 功能）
- 录制功能（非核心）
